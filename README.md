## 1. 产品概述

### 1.1 产品定位

音频调频器是一款面向普通用户的**四段图形化均衡器**桌面软件。通过简单的推子操作，用户可以实时调节麦克风输入音频的各频段强弱，让声音变得更好听。同时支持混响效果和录音功能，适用于**直播调音、语音聊天、在线授课、内容创作**等场景。

### 1.2 核心价值

- **简单直观**：不需要音频工程知识，拖动推子即可感受到音色变化
- **实时反馈**：所有调节即时生效，配合 Bypass 一键对比原始声音
- **独立运行**：单文件 EXE，无需安装 Python 或任何依赖

### 1.3 目标用户

| 用户类型 | 使用场景 |
|----------|----------|
| 主播 / UP 主 | 配合直播伴侣使用，美化麦克风音色 |
| 语音聊天用户 | 优化游戏语音、远程会议的音质 |
| 在线教师 | 提升授课录音的清晰度 |
| 音乐爱好者 | 欣赏音乐时的个性化均衡调节 |

## 2. 功能需求

### 2.1 四段均衡器 (EQ)

#### 2.1.1 频段定义

| 频段 | 频率范围 | 滤波器类型 | 中心/截止频率 | Q值 | 增益范围 | 颜色 |
|------|----------|-----------|-------------|------|----------|------|
| LF（低频） | 20Hz – 200Hz | Low Shelf | 200Hz | 0.707 | -24dB ~ +24dB | #ac3038 暗红 |
| MID LF（中低频） | 200Hz – 600Hz | Peaking | 400Hz | 1.0 | -24dB ~ +24dB | #ab3088 紫红 |
| MID HF（中高频） | 600Hz – 6kHz | Peaking | 3kHz | 1.0 | -24dB ~ +24dB | #3830ab 蓝紫 |
| HF（高频） | 6kHz – 16kHz | High Shelf | 6kHz | 0.707 | -24dB ~ +24dB | #30aba7 青蓝 |

#### 2.1.2 交互方式

- **拖动推子**：垂直滑块，步长 0.5dB，支持鼠标拖拽
- **精确输入**：推子下方数字输入框，可直接键入目标 dB 值，回车确认
- **Bypass 对比**：一键切换 EQ 效果开关，切换时推子面板变暗提示
- **独立静音**：一键关闭监听输出，不影响录音

### 2.2 混响效果 (Reverb)

| 属性 | 说明 |
|------|------|
| 算法 | Moorer/Schroeder：4 路反馈 Comb 滤波器 + 2 路串联 Allpass 扩散器 |
| 控制 | QDial 旋钮，范围 0% – 100% |
| 延迟线 | 29.7ms / 37.1ms / 41.1ms / 43.7ms（素数比例） |
| 扩散 | Allpass 5.0ms + 1.7ms |
| 处理 | 后 EQ 阶段，立体声独立处理 |
| 联动 | 反馈量随 Wet 比例自动调节 |

### 2.3 录音

| 属性 | 说明 |
|------|------|
| 触发 | 按钮点击开始/停止 |
| 格式 | 16-bit WAV，采样率与设备一致 |
| 输出 | 自动保存至桌面，命名格式 `录音_YYYYMMDD_HHmmss.wav` |
| 内容 | 录制 EQ + 混响处理后的最终音频。Bypass 启用时录制原始声音 |
| 计时 | 实时显示 `MM:SS` |

### 2.4 设备管理

| 功能 | 说明 |
|------|------|
| 输入设备选择 | 枚举系统所有麦克风/虚拟音频设备，下拉切换 |
| 输出设备选择 | 枚举系统所有扬声器/耳机，下拉切换 |
| 声道模式 | 立体声 / 单声道切换，单声道时左右声道取均值混音 |
| 设备刷新 | 热插拔后手动刷新列表 |

### 2.5 声道模式

| 模式 | 行为 |
|------|------|
| 立体声（默认） | 左右声道独立处理，保持空间信息 |
| 单声道 | 输入下混为单声道，应用于所有输出通道 |

## 3. 技术架构

### 3.1 技术栈

| 层 | 技术 | 说明 |
|----|------|------|
| 语言 | Python 3.9 | 跨平台解释型语言 |
| GUI 框架 | PySide6 (Qt 6) | LGPL 协议，原生控件 |
| 音频 I/O | sounddevice 0.5 | 基于 PortAudio 的低延迟音频流 |
| 数值计算 | numpy 2.0 | 音频缓冲区向量化运算 |
| EQ 滤波器 | 手工实现 Biquad | RBJ Audio EQ Cookbook 公式 |
| 混响 | 手工实现 Moorer | Comb + Allpass 滤波器组 |
| 录音输出 | Python wave 模块 | 标准库，16-bit PCM WAV |
| 打包 | PyInstaller 6.21 | 单文件 EXE，约 56MB |

### 3.2 音频处理链

```
麦克风输入
  │
  ▼
sounddevice InputStream
  │
  ▼
声道模式节点（立体声 / 单声道下混）
  │
  ├─→ [Bypass 路由] → 干声直通 ──────────────┐
  │                                            │
  └─→ EQ 链（LF → MID LF → MID HF → HF）       │
        │                                       │
        ▼                                       │
      混响（4 Comb + 2 Allpass）                 │
        │                                       │
        ▼                                       │
      Wet/Dry 混合 ←───────────────────────────┘
        │
        ▼
      静音检测
        │
        ▼
      OutputStream → 扬声器
        │
        ▼
      [录音中] → WAV 文件
```

### 3.3 Biquad 滤波器设计

采用 Robert Bristow-Johnson 的 Audio EQ Cookbook 标准公式：

```
A  = 10^(gain_dB / 40)
w0 = 2π × f0 / fs
α  = sin(w0) / (2 × Q)

Low Shelf:
  b0 = A × ((A+1) - (A-1)×cos(w0) + 2×√A×α)
  b1 = 2A × ((A-1) - (A+1)×cos(w0))
  b2 = A × ((A+1) - (A-1)×cos(w0) - 2×√A×α)
  a0 = (A+1) + (A-1)×cos(w0) + 2×√A×α
  a1 = -2 × ((A-1) + (A+1)×cos(w0))
  a2 = (A+1) + (A-1)×cos(w0) - 2×√A×α

High Shelf: 对称变换

Peaking:
  b0 = 1 + α×A
  b1 = -2×cos(w0)
  b2 = 1 - α×A
  a0 = 1 + α/A
  a1 = -2×cos(w0)
  a2 = 1 - α/A
```

每个滤波器维护 4 个状态变量（x1, x2, y1, y2），每个声道 4 个滤波器实例，支持增益实时更新（`set_gain` 重算系数）。

### 3.4 混响算法

```
输入 x[n]
  │
  ├───────────────┐
  │               │
  ▼               │
Comb 1 (29.7ms)   │
Comb 2 (37.1ms)   │ 并联反馈 Comb
Comb 3 (41.1ms)   │
Comb 4 (43.7ms)   │ 含低通阻尼
  │               │
  ├── 求和 × 0.25
  │
  ▼
Allpass 1 (5.0ms)
  │
  ▼
Allpass 2 (1.7ms)   串联 Allpass 扩散
  │
  ▼
输出 y[n] = x[n] × (1 - wet) + ap_output × wet
```

### 3.5 线程模型

| 线程 | 职责 |
|------|------|
| 主线程 (Qt) | GUI 事件循环，控件交互，样式渲染 |
| 音频回调线程 | sounddevice 回调，音频块处理（EQ + 混响 + 录音） |

音频参数（增益、bypass、静音、混响量）以普通 Python 变量存储，回调线程只读，GUI 线程写入，无需加锁。
